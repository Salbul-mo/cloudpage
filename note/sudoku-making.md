#스도쿠 만들기

## 규칙

### 1. 한 행에 1~9 까지 겹치는 숫자가 없어야 한다.


### 2. 한 열에 1~9 까지 겹치는 숫자가 없어야 한다.


### 3. 3 X 3 박스 안에 1~9 까지 겹치는 숫자가 없어야 한다.


### 4. 처음 공개된 숫자의 갯수는 16개 이상이어야 한다.
유일한 경우의 수가 정해지는 공개된 숫자의 최소 갯수가 16개로 수학적 증명이 되었다.

### 5. 되도록 30개 이하로 숫자를 공개하도록 한다. 
그러나 경우에 따라 40개 넘게 공개해야만 유일한 경우의 수가 정해지는 케이스도 있으므로 유의한다.

2. Map
const map1 = new Map();

map1.set("a", 1);
map1.set("b", 2);
map1.set("c", 3);

Map의 특징 : 키 값은 유일하다. 평균적으로 집합 내의 요소의 수에 따라 하위 선형인 접근 시간을 제공하는 맵을 구현해야 한다. 
=> 복잡성이 O(n) 보다 나은 경우 내부적으로 해시 테이블(O(1) 룩업), 검색 트리(O(log(N))) 또는 기타 데이터구조로 표현될 수 있다.

Object 와의 차이
Object 와 유사하지만 Object 의 경우 프로토타입으로 기본 키가 존재한다. Map 은 기본적으로 키를 포함하지 않는다.
Object 의 경우 임의의 키-값을 설정한 경우 프로토타입을 재정의하는 객체 주입 공격에 취약하다. Map의 경우에는 키-값 쌍이 안전하다.
Map 의 키 값으로는 모든 타입이 설정 가능하다. Object 의 경우 String 또는 Symbol 타입이어야 한다.
Map 의 키는 단순한 방법으로 정렬된다. 항목을 삽입한 순서대로 항목, 키 및 값을 반복한다. Object의 경우 정렬이 가능하지만 메커니즘이 다양하므로 가능한 순서 속성에 의존하지 않아야 한다.
Map 의 아이템 수는 size 속성에서 쉽게 가져올 수 있다. Object 의 아이템 수는 수동으로 결정해야 한다.
Map 은 Iterable(순회 가능) 이므로 직접 반복할 수 있다. Object 의 경우 직접 Iteration 이 아닌 별도의 Iteration Protocol을 구현하거나 Object.keys 혹은 Object.entries 를 사용한 for 루프를 사용한다.
Map 은 키-값의 변경이 빈번한 경우 성능이 조금 더 좋다. Object 의 경우 키-값의 빈번한 변경에 최적화되어 있지 않다.
Map 은 직렬화 또는 구문 분석에 기본적인 지원이 없어 JSON.stringify() 의 인자로 replacer 를 사용하여 직접 직렬화 및 구문분석을 구현하여야 한다. Object 의 경우 기본적인 직렬화와 구문 분석이 지원된다.






3. Tree
스도쿠 규칙에 따라 숫자 조합을 트리로 구성?

### 중복된 숫자가 없게 잘 생성되었는지 유효성 체크할 방법
1. for 문으로 일일이 비교하는 루프 생성

2. 스도쿠 규칙을 사용하여 각 구역을 미리 생성. 나머지를 배치하는 방법

1) 2X2 박스 4개의 꼭지점 숫자 세트와 정 가운데 박스를 둘러싼 박스의 숫자 세트가 동일하다.
2) 상단 5X5 박스 대각선 상에 위치하는 하단 4X4 박스의 숫자 조합은 1~9 한 세트를 제외하고 동일하다.

이 규칙을 이용해 빠르게 숫자 조합을 배치하고 나머지는 재귀 호출이나 다른 로직을 사용하여 채우는 방식 사용

### 그 외 추가적인 규칙
1. 사용자가 임의의 힌트를 배치하도록 권한 부여
2. 원하는 힌트 갯수를 정하고 시작하도록 권한 부여
3. 힌트 출력
4. 노트를 사용할 수 있도록 편의성 추가
